# 📁 Nextra内容组织完全指南

## 🏗️ 基础目录结构

Nextra采用基于文件系统的路由，目录结构直接映射到网站导航。

## 🎯 文件命名规则

### MDX文件命名
- 使用kebab-case：`my-page.mdx`
- 文件名成为URL：`my-page.mdx` → `/my-page`
- 特殊文件：`index.mdx` 是目录默认页面

### 目录命名
- 小写字母和短横线：`getting-started/`
- 语言目录：`zh/`、`en/` 等

## ⚙️ _meta.tsx 配置详解

_meta.tsx 是Nextra的核心配置文件，控制导航显示：

### 基础配置
```typescript
import type { MetaRecord } from 'nextra'

export default {
  index: {
    type: 'page',
    display: 'hidden',  // 隐藏首页
  },
  'getting-started': {
    title: '🚀 快速开始',
    type: 'page',
  },
  docs: {
    title: '📚 文档',
    type: 'page',
  },
} satisfies MetaRecord
```

### 高级配置选项
```typescript
export default {
  // 带排序的页面
  tutorial: {
    title: '教程',
    type: 'page',
    order: 1,           // 控制显示顺序
  },
  
  // 外部链接
  github: {
    title: 'GitHub ↗',
    type: 'page',
    href: 'https://github.com/your-repo',
    newWindow: true,
  },
  
  // 分隔符
  '---': {
    type: 'separator',
  },
  
  // 带主题的页面
  blog: {
    title: '博客',
    type: 'page',
    theme: {
      sidebar: false,   // 隐藏侧边栏
      toc: true,        // 显示目录
      timestamp: true,  // 显示时间戳
    },
  },
}
```

```
src/content/zh/
```

## 📋 最佳实践

### 1. 目录结构规划
- **扁平化原则**：避免过深的嵌套（建议最多3-4级）
- **逻辑分组**：按功能、主题或用户需求分组
- **一致性**：保持命名风格统一

### 2. 导航设计
- **清晰的层级**：用图标和缩进区分层级
- **有意义的标题**：标题要准确描述页面内容
- **合理排序**：把最重要的内容放在前面

### 3. 文件组织
- **index.mdx**：每个目录都应该有首页
- **_meta.tsx**：每个目录都需要导航配置
- **命名规范**：使用kebab-case命名法
