# 📄 Frontmatter完全指南

Frontmatter是位于MDX/Markdown文件开头的**元数据区域**，使用YAML格式定义。

## 🔍 什么是Frontmatter？

Frontmatter就是文件开头被 `---` 包围的区域：

```yaml
---
title: "页面标题"
description: "页面描述"
date: "2024-12-19"
---
```

## 🎯 常用的Frontmatter字段

### 基础信息
- **title**: 页面标题
- **description**: 页面描述  
- **date**: 发布/创建日期

### 分类和组织
- **tags**: 标签数组
- **category**: 分类
- **author**: 作者信息

### 控制字段
- **published**: 是否发布
- **featured**: 是否特色内容
- **draft**: 是否为草稿

### SEO和显示
- **readingTime**: 预计阅读时间
- **lastModified**: 最后修改时间

## 🔧 Frontmatter的作用

### 1. 页面元数据
```yaml
---
title: "我的博客文章"
description: "这是一篇关于React的文章"
---
```
→ 用于生成 `<title>` 和 `<meta>` 标签

### 2. 导航和排序
```yaml
---
title: "第一章：基础"
order: 1
sidebar: true
---
```
→ 控制页面在导航中的位置和显示

### 3. 内容筛选
```yaml
---
tags: ["React", "TypeScript"]
category: "前端"
published: true
---
```
→ 用于内容分类和筛选

## 💡 实际应用场景

### 博客文章
```yaml
---
title: "MDX进阶技巧"
date: "2024-12-19"
author: "张三"
tags: ["MDX", "React"]
readingTime: "8分钟"
---
```

### 文档页面
```yaml
---
title: "API参考"
sidebar: true
order: 3
toc: true
---
```

### 产品页面
```yaml
---
title: "产品介绍"
description: "我们的主打产品"
featured: true
price: "$99"
---
```

## 🚀 高级用法

### 1. 数组和对象
```yaml
---
tags: ["标签1", "标签2", "标签3"]
author:
  name: "张三"
  email: "zhangsan@example.com"
  avatar: "/images/avatar.jpg"
---
```

### 2. 布尔值和数字
```yaml
---
published: true
featured: false
order: 5
price: 99.99
---
```

### 3. 多行文本
```yaml
---
description: |
  这是一个很长的描述
  可以跨越多行
  保持格式
---
```

## 🎨 最佳实践

1. **保持一致性**: 在项目中使用统一的字段命名
2. **必填字段**: 至少包含title
3. **有意义的描述**: description要准确描述页面内容
4. **合理使用标签**: tags帮助内容分类
5. **日期格式**: 使用ISO格式的日期
